# Bars AI — Платформа обучения

## Wiki (LLM Wiki Pattern)

Проект использует Obsidian vault в `docs/` как **LLM-managed wiki**. LLM пишет и поддерживает вики, человек направляет и проверяет.

### Структура

```
docs/
├── Главная.md          # Навигационный хаб для Obsidian
├── index.md            # Машинный индекс — LLM читает первым для поиска страниц
├── log.md              # Хронологический лог изменений (append-only)
├── архитектура/         # Обзор системы, структура проекта, БД
├── api/                # Эндпоинты, аутентификация, AI сервисы
├── фронтенд/           # Роутинг, сторы, компоненты, Python Coding
├── бэкенд/             # Модели, сервисы, конфигурация
├── фичи/               # Фичи продукта: геймификация, спринты, курсы, онбординг, i18n, admin
├── гайды/              # Быстрый старт, добавление курса, деплой
└── superpowers/        # Планы и спеки (исторические, не трогать)
```

### Ключевые файлы

- **`docs/index.md`** — индекс всех страниц с описаниями. LLM читает его первым чтобы найти нужные страницы. Обновляй при создании/удалении страниц.
- **`docs/log.md`** — append-only лог. Каждая запись: `## [YYYY-MM-DD] action | Description`. Действия: `ingest`, `fix`, `refactor`, `lint`.
- **`docs/Главная.md`** — человекочитаемый хаб для навигации в Obsidian.

### Workflows

#### Ingest (после каждой фичи/изменения)
1. Обнови затронутые wiki-страницы (маппинг ниже)
2. Обнови `docs/index.md` если добавил/удалил страницу
3. Добавь запись в `docs/log.md`
4. Используй `[[вики-линки]]` для связей между заметками

#### Query (поиск информации)
1. Прочитай `docs/index.md` чтобы найти релевантные страницы
2. Прочитай найденные страницы
3. Если ответ стоит сохранить — создай новую страницу в wiki

#### Lint (периодическая проверка)
Проверяй wiki на: устаревшие данные, отсутствующие перекрёстные ссылки, упомянутые но не задокументированные сущности, противоречия между страницами.

### Маппинг файлов → wiki-страниц

| Изменённые файлы | Wiki-страницы |
|---|---|
| `backend/app/routers/*` | `архитектура/Структура проекта.md`, `api/API Обзор.md` |
| `backend/app/models/*` | `бэкенд/Модели.md`, `архитектура/База данных.md` |
| `backend/app/services/*` | `бэкенд/Сервисы.md` |
| `src/pages/*`, `src/App.tsx` | `фронтенд/Роутинг.md` |
| `src/store/*` | `фронтенд/Сторы.md` |
| `src/components/*` | `фронтенд/Компоненты.md` |
| `src/lib/landing-i18n.ts` | `фичи/Landing i18n.md` |
| `src/pages/Onboarding.tsx` | `фичи/Онбординг.md` |
| `backend/app/config.py` | `бэкенд/Конфигурация.md` |
| `admin/src/*` | `фичи/Admin Panel.md` |
| `Dockerfile`, `.railwayignore` | `фичи/Production.md` |
| Новая фича продукта | Создай страницу в `фичи/` |

### Правила

- **LLM владеет wiki** — пиши, обновляй, поддерживай перекрёстные ссылки
- **Документация обязательна** — после каждой фичи обновляй wiki и лог
- **`[[вики-линки]]`** обязательны для связей между заметками
- **Не трогай `docs/superpowers/`** — это исторические планы/спеки
- Цель: любой разработчик понимает проект, читая только `docs/`
