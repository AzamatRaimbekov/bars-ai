# Онбординг

Страница `src/pages/Onboarding.tsx`. Показывается новым пользователям до перехода на дашборд. Флаг `User.onboarding_complete` переключается в `true` при завершении.

## 4-шаговый поток

Прогресс отображается четырьмя точками в верхней части экрана (активная растягивается).

### Шаг 0 — Приветствие и имя

Маскот Барсбек с речевым пузырём. Если имя пользователя не задано — предлагается поле ввода. Переход по нажатию кнопки или Enter. При изменении имени вызывается `updateUser({ name })`.

### Шаг 1 — Выбор интересов

Загружаются теги курсов через `courseApi.getTags()`. Отображаются как кнопки-пилюли. Можно выбрать несколько. Требуется хотя бы один тег для перехода дальше.

### Шаг 2 — Мини-чат

Два последовательных вопроса от Барсбека (`onboarding.chatQ1`, `onboarding.chatQ2`). Ответы сохраняются в `chatAnswers[]`. Есть кнопка «Пропустить». Ответы передаются в `assessment_context` при завершении.

### Шаг 3 — Рекомендации курсов

Вызывается `courseApi.recommend(selectedTags)`. Отображается список курсов с возможностью сразу записаться. Кнопка «Начать обучение» сохраняет `interests`, `onboarding_complete: true` и `assessment_context`, затем перенаправляет на `/dashboard`.

## Финальное сохранение

```ts
updateUser({
  interests: selectedTags,
  onboarding_complete: true,
  assessment_context: chatAnswers.join(' | '),
})
```

После сохранения вызывается `fetchUser()` для обновления стора.
