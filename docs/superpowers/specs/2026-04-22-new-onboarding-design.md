# Новый онбординг с маскотом Barsbek

**Дата:** 2026-04-22
**Подход:** Гибрид (визуальные шаги + короткий чат), фильтрация курсов по тегам

## Проблема

Текущий онбординг привязан к 4 фиксированным направлениям (frontend, english, callcenter, cib) и не помогает пользователю найти конкретные курсы из маркетплейса. Нужно заменить его на более гибкий, который подбирает реальные курсы из БД по интересам пользователя.

## Флоу (4 шага)

### Шаг 1 — Приветствие Barsbek
- Маскот Barsbek (из существующих assets) с речевым пузырём: "Салам! Я Barsbek, помогу найти курсы для тебя"
- Поле ввода имени (если ещё нет в профиле)
- Кнопка "Поехали"

### Шаг 2 — Выбор интересов
- Barsbek сверху с пузырём: "Что тебе интересно? Выбери всё что нравится"
- Сетка тегов-чипов (мульти-выбор) — теги берутся из уникальных тегов/категорий курсов в БД
- Минимум 1 тег для продолжения
- Анимированное выделение выбранных тегов (оранжевый градиент)

### Шаг 3 — Короткий чат с Barsbek
- Barsbek задаёт 1-2 вопроса:
  1. "Какой у тебя опыт в выбранных темах?"
  2. "Учишься для себя или для работы?"
- Ответы сохраняются в профиль (`assessment_context`), но НЕ влияют на фильтрацию курсов
- Кнопка "Пропустить" — чат необязателен

### Шаг 4 — Рекомендации курсов
- Barsbek: "Вот что я подобрал для тебя!"
- Карточки курсов, отфильтрованные по тегам из шага 2
- Пользователь может записаться (enroll) на курсы прямо из онбординга
- Кнопка "Начать обучение" → переход на /dashboard

## Изменения в бэкенде

### Модель Course — добавить теги
- Поля `tags` нет — есть только `category: str`. Нужно добавить:
- `tags: Mapped[list | None] = mapped_column(JSON, nullable=True, default=list)` — JSON array строк
- Alembic миграция
- Фильтрация: `GET /api/courses/recommend?tags=...` ищет курсы где хотя бы один тег пересекается

### Модель User — новые поля
- `interests: list[str]` — выбранные теги интересов (JSON array)
- `onboarding_complete: bool` — флаг завершения онбординга (default: False)
- `assessment_context: str | None` — ответы из чата с Barsbek (опционально)
- Alembic миграция
- Поле `direction` остаётся для обратной совместимости, но больше не используется в онбординге

### Новые эндпоинты
- `GET /api/courses/tags` — список уникальных тегов из всех опубликованных курсов
- `GET /api/courses/recommend?tags=frontend,python` — курсы, отфильтрованные по тегам (пересечение)

### Обновить эндпоинт
- `PATCH /api/users/me` — принимать новые поля: `interests`, `onboarding_complete`, `assessment_context`
