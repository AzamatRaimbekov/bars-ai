# Сторы (Zustand)

Zustand 5 для управления состоянием. Все сторы в `src/store/`.

## authStore.ts
- `user` — текущий пользователь (`UserData`)
- `isAuthenticated` — флаг авторизации
- `isLoading` — загрузка состояния
- `login()` / `register()` / `logout()`
- `fetchUser()` — загрузка актуальных данных пользователя
- `updateUser(data)` — PATCH `/users/me`; принимает поля `name`, `language`, `avatar_url`, `direction`, `assessment_level`, `interests`, `onboarding_complete` и опциональный `assessment_context`
- `tryRestore()` — восстановление сессии из httpOnly cookie / sessionStorage

### UserData
| Поле | Тип | Описание |
|---|---|---|
| `id` | `string` | UUID пользователя |
| `email` | `string` | Email |
| `name` | `string` | Отображаемое имя |
| `direction` | `string` | Направление обучения |
| `assessment_level` | `string` | Уровень по результатам ассессмента |
| `language` | `string` | Язык интерфейса |
| `avatar_url` | `string \| null` | Ссылка на аватар |
| `xp` | `number` | Накопленный XP |
| `level` | `string` | Игровой уровень |
| `streak` | `number` | Текущая серия дней |
| `longest_streak` | `number` | Рекорд серии |
| `completed_nodes` | `string[]` | Пройденные узлы роадмапа |
| `completed_lessons` | `string[]` | Пройденные уроки |
| `earned_badges` | `string[]` | Полученные бейджи |
| `role` | `string` | Роль (`student`, `admin`, …) |
| `interests` | `string[]` | Теги интересов (онбординг) |
| `onboarding_complete` | `boolean` | Завершён ли онбординг |

## chatStore.ts
- Хранение сообщений чата с AI-ментором
- Отправка/получение сообщений

## roadmapStore.ts
- Состояние роадмапа (React Flow)
- Ноды, рёбра, прогресс

## simulatorStore.ts
- Состояние симулятора собеседований
- Вопросы, ответы, оценки

## userStore.ts
- Дополнительные данные пользователя
- Настройки, предпочтения

## Админ-панель (admin/src/store/)

### authStore.ts
- `token` / `adminName` / `isAuthenticated`
- `login(email, password)` — POST /api/auth/login, сохраняет токен в sessionStorage
- `logout()` — очищает sessionStorage, редирект на /login
- `hydrate()` — восстановление сессии из sessionStorage
