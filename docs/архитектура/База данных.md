# База данных

## Модели (SQLAlchemy)

### User (`user.py`)
Основная модель пользователя — аутентификация, профиль, прогресс.

Новые поля (онбординг):
- `interests: JSON` (nullable) — список интересов пользователя, выбранных при онбординге
- `onboarding_complete: Boolean` (default False) — флаг завершения онбординга (заменяет проверку по `direction`)
- `assessment_context: Text` (nullable) — контекст оценки уровня пользователя

### Course (`course.py`)
Курсы с модулями и уроками. Связи с прогрессом.
Поле `tags: JSON` (nullable) — список тегов для фильтрации курсов по интересам пользователя (онбординг).

### Progress (`progress.py`)
Прогресс пользователя по урокам, XP, уровни.

### Badge (`badge.py`)
Бейджи и достижения пользователей.

### Sprint (`sprint.py`)
Спринты (еженедельные челленджи), трофеи за выполнение.

### Mentor (`mentor.py`)
История чата с AI-ментором.

### PaymentRequest (`payment.py`)
Запросы на оплату курсов. Пользователь загружает скриншот оплаты, админ подтверждает/отклоняет. Статусы: pending, approved, rejected.

### RefreshToken (`refresh_token.py`)
JWT refresh-токены для безопасной аутентификации.

### PasswordResetToken (`user.py`)
Одноразовые токены сброса пароля. Поля: `token_hash`, `expires_at` (1 час), `used: bool`. При запросе сброса все старые активные токены инвалидируются.

## Миграции

Используется **Alembic** с async-поддержкой.

```bash
# Создать миграцию
cd backend
alembic revision --autogenerate -m "описание"

# Применить миграции
alembic upgrade head
```

## Подключение

- Драйвер: `asyncpg` (async PostgreSQL)
- Engine: `SQLAlchemy 2.0` с `AsyncSession`
- Pool: стандартный пул asyncpg
