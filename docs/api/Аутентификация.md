# Аутентификация

## Схема

```
Клиент → POST /api/auth/login → { access_token, refresh_token }
                                        │
                        ┌────────────────┴─────────────────┐
                        │                                  │
                   access_token                      refresh_token
                   (30 мин)                          (7 дней)
                        │                                  │
              Authorization: Bearer <token>      POST /api/auth/refresh
```

## JWT-токены

- **Access token** — время жизни 30 минут (`JWT_ACCESS_EXPIRE_MINUTES`)
- **Refresh token** — время жизни 7 дней (`JWT_REFRESH_EXPIRE_DAYS`)
- Подписываются `JWT_SECRET` через `python-jose`
- Хеширование паролей: `passlib` + `bcrypt`
- **B2B расширение**: JWT payload включает `org_id` и `is_superadmin`

## CurrentUser Dependency

`backend/app/dependencies.py` — `get_current_user` возвращает полный объект User (не только sub из JWT). Используется для B2B проверок (organization_id, is_superadmin).

## Permissions (RBAC)

`backend/app/permissions.py` — список разрешений и dependency `require_permission()`. Проверяет union прав: superadmin > org_admin > role permissions.

## Фронтенд AuthGuard

`src/App.tsx` оборачивает защищённые маршруты в `<AuthGuard>`:

1. Проверяет `isAuthenticated` из `authStore`
2. Если не авторизован → редирект на `/login`
3. Если не прошёл onboarding (нет `direction`) → редирект на `/`
4. Если прошёл onboarding и на `/` → редирект на `/dashboard`

## Публичные маршруты

- `/login` — вход
- `/register` — регистрация
- `/courses` — каталог курсов (без авторизации)
- `/courses/:id` — детали курса (без авторизации)
